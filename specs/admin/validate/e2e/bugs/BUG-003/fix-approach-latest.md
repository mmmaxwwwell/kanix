The `_addPhoto()` method in `_FileClaimFormState` (`customer/lib/screens/warranty_screen.dart:287`) was a stub that unconditionally appended fake filenames (`'photo_N.jpg'`) to the `_photoNames` list without opening a file picker. Fixed by: (1) adding `image_picker: ^1.1.2` to `customer/pubspec.yaml`; (2) importing `package:image_picker/image_picker.dart` in `warranty_screen.dart`; (3) rewriting `_addPhoto()` as an async method that calls `ImagePicker().pickImage(source: ImageSource.gallery)` — only updating `_photoNames` with `image.name` when the picker returns a non-null result. Photos sent to the API now reflect real filenames from the user's gallery, not fabricated strings.
