Added a `POST /api/test/seed-paid-order` endpoint to `api/src/server.ts`, gated to non-production environments (`config.NODE_ENV !== "production"`). The endpoint creates a product + variant, reuses or creates an inventory location with 50 units available, creates a cart, adds the variant, calls `createCheckoutOrder` with a fake Stripe payment intent ID (no real Stripe call), then calls `handlePaymentSucceeded` to simulate the `payment_intent.succeeded` webhook — transitioning the order to `confirmed` + `paid` and auto-creating a fulfillment task. Returns `{ order_id, order_number, payment_status: "paid" }` with HTTP 201. Also imported `inventoryLocation` and `inventoryBalance` schemas into server.ts to support the direct inventory insert.
