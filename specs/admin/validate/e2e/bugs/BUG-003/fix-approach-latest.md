Replaced `window.__KANIX_API_URL__` in `site/src/lib/cart.ts` with `import.meta.env.PUBLIC_API_URL`. The original code referenced a window global that was never injected (the `vite.define` config was missing from `astro.config.mjs`). Using `import.meta.env.PUBLIC_API_URL` is the idiomatic Astro/Vite pattern already used by `products.ts` and `kits.ts` — Vite statically replaces it at build time in both server and client code. Combined with the BUG-001 env var fix, this ensures `getApiUrl()` returns the correct API URL for all cart/checkout operations.
