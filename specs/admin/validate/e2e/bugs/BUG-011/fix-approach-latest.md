Modified `createShippingAdapter()` in `api/src/services/shipping-adapter.ts` to detect placeholder API keys containing "REPLACE_ME" and fall back to the stub adapter. The factory previously only checked for empty string and "test-key", so the `.env.example` placeholder "EZAK_REPLACE_ME" was treated as a real key, causing EasyPost API calls to fail with invalid credentials. The fix adds `!key.includes("REPLACE_ME")` to the guard condition.
