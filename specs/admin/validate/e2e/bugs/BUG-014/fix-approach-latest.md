Modified `scripts/stripe-listen-start.sh` and `scripts/stripe-webhook-secret.sh` to load `STRIPE_SECRET_KEY` from `.env` and pass it via `--api-key` to all `stripe listen` invocations. When a valid key is available (not containing "REPLACE_ME"), the scripts skip the `stripe login` check entirely. This enables headless/CI usage. The `stripe login` workflow still works as a fallback when no API key is set.
