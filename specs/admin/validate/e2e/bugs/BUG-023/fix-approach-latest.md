Added a "Submit Bundle" `OutlinedButton.icon` and `_submitBundle` method to `_DisputeDetailContent` in `admin/lib/screens/disputes_screen.dart`. The button appears alongside the existing Generate Bundle button when `!dispute.evidenceSubmitted && dispute.status == 'needs_response'`. Clicking it shows a confirmation dialog then calls `POST /api/admin/disputes/$disputeId/submit-bundle`, refreshes the dispute detail and evidence providers, and shows a success/error snackbar. The API endpoint was already fully implemented; this was purely a missing UI affordance.
