Fixed the cart item removal bug by addressing two issues. First, in `site/src/lib/cart.ts`, the `apiRequest` function unconditionally set `Content-Type: application/json` on every request, including bodyless DELETE requests. Fastify v5 invokes its JSON body parser when it sees this header on DELETE, and since there's no body, parsing fails with a 400 error. The fix conditionally sets the header only when `options.body` is present. Second, in `site/src/pages/checkout.astro`, the remove button's catch block silently swallowed the error (`// ignore remove errors`), masking the 400 failure. Changed it to log the error with `console.error` for debuggability.
