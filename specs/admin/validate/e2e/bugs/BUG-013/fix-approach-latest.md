Two changes in `site/src/pages/checkout.astro`: (1) Extracted the submit handler body into a standalone `handleCheckoutSubmit()` function, then attached it to both the form's `submit` event and the button's `click` event. This ensures the checkout logic fires regardless of how the button click is dispatched (native vs programmatic/automation). (2) Narrowed the `validateForm()` error reset selector from `[id$='-error']` to `#shipping-form [id$='-error']:not(#checkout-error):not(#stripe-card-error)` so it no longer inadvertently hides the checkout error div or Stripe card error.
