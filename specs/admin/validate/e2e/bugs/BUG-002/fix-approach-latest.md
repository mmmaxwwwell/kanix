The customer app's `AppShell` (`customer/lib/widgets/app_shell.dart`) had only 4 bottom nav destinations: Catalog, Cart, Orders, Account. The `/support` and `/warranty` routes existed in `router.dart` but were unreachable from the nav bar because no Support tab was present. Fixed by: (1) adding a Support `NavigationDestination` (icon: `Icons.support_agent_outlined`) at index 2 between Cart and Orders; (2) inserting `'/support'` into the `_routes` list at index 2 so tapping the Support tab navigates to `/support`; (3) updating the `/contributor` index mapping from `return 3` to `return 4` since Account shifted from index 3 to index 4. The existing `_selectedIndex` check `if (location.startsWith('/warranty')) return 2` was already correct — it now correctly highlights the Support tab when viewing warranty routes.
