Updated `Dispute.fromJson` in `admin/lib/models/dispute.dart` to use the correct API field names: `providerDisputeId` (was `stripeDisputeId`), `openedAt` (was `createdAt`), `dueBy` (was `evidenceDueBy`), and derived `evidenceSubmitted` from `(json['evidenceCount'] as int? ?? 0) > 0` (was `json['evidenceSubmitted'] as bool?`). For `updatedAt`, which is not returned by the API, fall back to `openedAt`. Also made `reason` null-safe with an empty string default. These are direct field name renames to match the actual API contract confirmed by `GET /api/admin/disputes`.
