Added a delete of `cart_kit_selection` rows before deleting the `cart_line` row in `api/src/db/queries/cart.ts:removeCartItem()`. The `cart_kit_selection` table has a foreign key (`fk_cks_cart_line`) referencing `cart_line(id)` with `ON DELETE RESTRICT`, which blocked deletion of kit cart lines. The fix deletes any child `cart_kit_selection` rows for the given `cartLineId` before attempting the parent `cart_line` delete. For non-kit items this is a no-op (0 rows affected). The `cartKitSelection` import was already present in the file.
