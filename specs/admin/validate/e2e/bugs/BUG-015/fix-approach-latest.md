Hardened the shipping adapter factory guard in `api/src/services/shipping-adapter.ts:376` to also detect keys containing "placeholder" (in addition to the existing "REPLACE_ME" check). The previous fix changed the E2E setup script's default EASYPOST_API_KEY from `EZAK_e2e_placeholder_key` to `EZAK_REPLACE_ME`, but the running API server wasn't restarted, so it still used the old key. By adding `!key.includes("placeholder")` to the factory's stub-detection guard, any placeholder-style key (including the old `EZAK_e2e_placeholder_key`) now correctly triggers the stub adapter, preventing the dynamic import of the uninstalled `@easypost/api` package.
