Changed the default EASYPOST_API_KEY in `test/e2e/setup.sh` from `EZAK_e2e_placeholder_key` to `EZAK_REPLACE_ME`. The factory in `shipping-adapter.ts` checks for `REPLACE_ME` in the key to decide whether to return the stub adapter. The old placeholder bypassed that guard, causing the real EasyPost adapter to be constructed, which then failed with ERR_MODULE_NOT_FOUND because `@easypost/api` is intentionally not installed. The new value matches the `.env.example` convention and correctly triggers the stub path.
