Fixed the `ANDROID_BUILD_ROOT=customer` misconfiguration in `.mcp.json` — changed to `admin` since all E2E admin tests target the admin Flutter app, not the customer app. This ensures mcp-android resolves APK and build paths correctly for the admin app. The deeper root cause — the runner's `ensure_runtime()` in `parallel_runner.py` skips the emulator liveness check when `_booted=True` from a prior spawn, so if the Android emulator goes offline between spawns (OOM-kill, crash) the mcp-android MCP server starts against a dead emulator and the executor session sees zero MCP tools — requires a fix in `parallel_runner.py` (read-only in the agent sandbox). The proposed runner fix adds an `adb shell getprop sys.boot_completed` liveness check in `ensure_runtime` before taking the early-return shortcut; if the emulator is offline it tears down the stale runtime entry and falls through to a fresh boot. See `proposed_diff` in the out-of-scope claim for the exact change needed.
