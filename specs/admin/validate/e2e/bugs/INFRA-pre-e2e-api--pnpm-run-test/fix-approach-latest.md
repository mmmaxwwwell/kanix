Excluded `*.integration.test.ts` files from the default `pnpm test` (vitest run) command by adding them to the `exclude` array in `api/vitest.config.ts`. Integration tests require DATABASE_URL and live services (PostgreSQL, SuperTokens) which are not available in the default test environment. Added a dedicated `pnpm test:integration` script backed by a new `vitest.integration.config.ts` that includes only integration tests, so they can be run explicitly when services are available via `process-compose up`.
