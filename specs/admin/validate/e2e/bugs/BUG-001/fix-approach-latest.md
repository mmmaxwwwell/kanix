Fixed the root cause in `admin/lib/models/order.dart`: the `listAllOrders` API query only returns 9 fields (`id`, `orderNumber`, `email`, `status`, `paymentStatus`, `fulfillmentStatus`, `totalMinor`, `placedAt`, `createdAt`) but `Order.fromJson` was doing unsafe `null as String` casts on two fields absent from the list response: `shippingStatus` (line 55: `json['shippingStatus'] as String`) and `updatedAt` (line 70: `json['updatedAt'] as String`). Both crash at runtime with `type 'Null' is not a subtype of type 'String' in type cast` whenever any order in the paginated response lacks those fields (which is every order in the list view). The fix: (1) `shippingStatus` deserialization changed to `json['shippingStatus'] as String? ?? 'not_shipped'` — preserves non-nullable field type so no UI changes needed; (2) `updatedAt` field changed to `DateTime?`, constructor parameter made optional, and `fromJson` made null-safe with a null check before `DateTime.parse`. Rebuilt APK with `flutter clean && flutter build apk --debug` to bust Gradle incremental cache; reinstalled on device.
